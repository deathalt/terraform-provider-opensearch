# opensearch_dashboard_data_source (Resource)

Provides an OpenSearch Dashboards data source connection resource.

This resource uses the OpenSearch Dashboards API, so the provider must be configured with `dashboards_url`.

## Example Usage

```terraform
provider "opensearch" {
  url            = "http://127.0.0.1:9200"
  dashboards_url = "http://127.0.0.1:5601"
}

resource "opensearch_dashboard_data_source" "remote" {
  title       = "Remote cluster"
  endpoint    = "https://remote.example.com"
  tenant_name = "tenant-a"
  auth_type   = "no_auth"
}
```

## Username and Password

```terraform
resource "opensearch_dashboard_data_source" "remote" {
  title     = "Remote cluster"
  endpoint  = "https://remote.example.com"
  auth_type = "username_password"

  username = "admin"
  password = var.remote_cluster_password
}
```

## AWS SigV4

```terraform
resource "opensearch_dashboard_data_source" "remote" {
  title     = "Remote AWS domain"
  endpoint  = "https://search-example.us-east-1.es.amazonaws.com"
  auth_type = "sigv4"

  aws_access_key = var.aws_access_key
  aws_secret_key = var.aws_secret_key
  aws_region     = "us-east-1"
  aws_service    = "es"
}
```

## Associate with a Workspace

Use the same `tenant_name` on the data source, workspace, and workspace association resource.

```terraform
resource "opensearch_dashboard_workspace" "logs" {
  name        = "Logs"
  features    = ["use-case-observability"]
  tenant_name = "tenant-a"
}

resource "opensearch_dashboard_data_source" "remote" {
  title       = "Remote cluster"
  endpoint    = "https://remote.example.com"
  tenant_name = "tenant-a"
  auth_type   = "no_auth"
}

resource "opensearch_dashboard_workspace_objects" "remote_data_source" {
  workspace_id = opensearch_dashboard_workspace.logs.id
  tenant_name  = "tenant-a"

  saved_object {
    type = "data-source"
    id   = opensearch_dashboard_data_source.remote.id
  }
}
```

## Schema

### Required

- `auth_type` (String) Authentication type. Supported values are no_auth, username_password, and sigv4.
- `endpoint` (String) Data source connection endpoint URL.
- `title` (String) Data source connection title.

### Optional

- `aws_access_key` (String, Sensitive) AWS access key for sigv4 authentication.
- `aws_region` (String) AWS region for sigv4 authentication.
- `aws_secret_key` (String, Sensitive) AWS secret key for sigv4 authentication.
- `aws_service` (String) AWS SigV4 service name. Use es for OpenSearch or aoss for OpenSearch Serverless.
- `description` (String) Data source connection description.
- `password` (String, Sensitive) Password for username_password authentication.
- `tenant_name` (String) OpenSearch Dashboards tenant name where the data source connection is stored. Defaults to the global tenant.
- `username` (String, Sensitive) Username for username_password authentication.

### Read-Only

- `data_source_engine_type` (String) Detected data source engine type.
- `data_source_version` (String) Detected data source version.
- `id` (String) Data source saved object ID.
- `installed_plugins` (Set of String) Detected plugins installed on the data source.

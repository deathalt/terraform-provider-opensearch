# opensearch_dashboard_workspace_objects (Resource)

Associates existing saved objects with an OpenSearch Dashboards workspace.

This resource uses the OpenSearch Dashboards API, so the provider must be configured with `dashboards_url`.

The workspace and every saved object must exist in the same OpenSearch Dashboards tenant. If `tenant_name`
is omitted, the global tenant is used.

## Example Usage

```terraform
provider "opensearch" {
  url            = "http://127.0.0.1:9200"
  dashboards_url = "http://127.0.0.1:5601"
}

resource "opensearch_dashboard_workspace" "logs" {
  name        = "Logs"
  description = "Production log analysis"
  features    = ["use-case-observability"]
  tenant_name = "tenant-a"
}

resource "opensearch_dashboard_workspace_objects" "logs_index_pattern" {
  workspace_id = opensearch_dashboard_workspace.logs.id
  tenant_name  = "tenant-a"

  saved_object {
    type = "index-pattern"
    id   = "logs-*"
  }
}
```

## Workspace Mapping

OpenSearch Dashboards stores workspace associations in the saved object's top-level `workspaces` field.
If association fails with `strict_dynamic_mapping_exception` for `workspaces`, the tenant's Dashboards saved
objects index was likely created before workspace mappings were enabled or before migrations completed. Run
OpenSearch Dashboards migrations with workspaces enabled, or update the tenant saved objects index mapping to
include `workspaces` as a `keyword` field.
